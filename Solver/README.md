# Faculty Scheduling Solver

A constraint-based faculty scheduling tool built on [Google OR-Tools CP-SAT](https://developers.google.com/optimization/reference/python/sat/python/cp_model). Given faculty preferences, section time blocks, and workload limits, it finds the optimal faculty-to-section assignment that maximises preference scores while satisfying all constraints.

## Input CSV Formats

### `preferences.csv`
Rows are sections identified by CRN, columns are faculty names.

| section_id | Faculty A | Faculty B | Faculty C |
|---|---|---|---|
| 76192 | 3 | x | 1 |
| 70722 | 0 | 2 | x |

**Score values:**
- `0–3` — preference score (0 = neutral, 3 = strongly prefer)
- `x` — hard exclusion, this faculty **cannot** be assigned to this section regardless of score

The difference between `0` and `x` matters — a score of `0` means the faculty is indifferent but the solver can still assign them if needed. An `x` is an absolute ban the solver can never override.

---

### `time_blocks.csv`
One row per section. Online sections (no time) should still be included with blank time fields.

| CRN | Sub | Days | Start Time | End Time | Room |
|---|---|---|---|---|---|
| 76192 | CSC | TuTh | 13:00 | 14:15 | East Hall 010 |
| 70722 | CSC | MWF | 14:00 | 14:50 | East Hall 003 |

**Supported day patterns and their fixed durations:**

| Pattern | Days |
|---|---|
| `MWF` | Mon / Wed / Fri |
| `TuTh` or `TTh` | Tue / Thu |
| `MW` | Mon / Wed |
| `MTuWF` | Mon / Tue / Wed / Fri |
| `M`, `Tu`, `W`, `Th`, `F` | Single-day — bucketed separately, not sent to OR |

End time is read directly from the file. Sections with no Days value are treated as internet sections.
The `room` column is reserved for future use — leave it blank for now.

---


### `sections.csv` — Optional

One row per section. If this file is missing, section metadata is derived from `time_blocks.csv`. The `Faculty` column is informational only — the solver assigns faculty from `preferences.csv` regardless of what is listed here.

| CRN | Sub | Num | Seq | Crd | Desc | Seats | Waitlist | Days | Time | Room | Faculty | Current Workload |
|---|---|---|---|---|---|---|---|---|---|---|---|---|
| 76192 | CSC | 150 | D01 | 3 | Computer Science I | 18/25 | 0 | TuTh | 1300-1415 | East Hall 010 | | 5 |

- `Seats` — combined format `current/max` (e.g. `18/25`)
- `Time` — combined format `HHMM-HHMM` (e.g. `1300-1415`)
- `Current Workload` — workload units this section contributes to a faculty member's total load. Defaults to `DEFAULT_COURSE_WORKLOAD` if missing (configured at the top of `faculty_scheduling.py`).

---

### `workload.csv` — Optional

One row per faculty member who has pre-assigned research or buyout units. Faculty not listed default to 0 research units, meaning their full cap is available.

| Faculty | Research Units |
|---|---|
| Wendy Simmermon | 10 |
| Jeffrey E Elbert | 15 |

The effective cap for each faculty member is:

```
effective cap = FACULTY_MAX_WORKLOAD - research_units
```

---
## Workload Configuration
Workload limits are configured via constants at the top of `faculty_scheduling.py`:
```python
FACULTY_MAX_WORKLOAD  = 30   # soft cap — total units per faculty per semester
OVERLOAD_PENALTY      = 10   # preference points lost per unit over cap
DEFAULT_COURSE_WORKLOAD = 4  # fallback units per section when current_workload is missing
```

The workload constraint is **soft** — the solver penalises exceeding the cap in the objective function but will do so if no feasible solution exists within the cap.

## How to Run

### Full pipeline — uses default CSV filenames in the same folder
```bash
python faculty_scheduling.py
```
### Validate an existing schedule
```bash
python faculty_scheduling.py --validate schedule.csv
```

### Custom file paths
```bash
python faculty_scheduling.py \
  --sections     data/sections.csv \
  --time_blocks  data/time_blocks.csv \
  --preferences  data/preferences.csv \
  --workload     data/workload.csv \
  --output       output/schedule.csv
```

### Custom log output path
```bash
python faculty_scheduling.py --log my_log.txt
```

### Without sections.csv — derived from time_blocks.csv
```bash
python faculty_scheduling.py --time_blocks data/time_blocks.csv --preferences data/preferences.csv
```

### Run the test suite — no CSV files needed
```bash
python faculty_scheduling.py --test
```

## Output

The solver writes `schedule.csv` — crn,assigned_faculty

```
crn,assigned_faculty
76192,Wendy Simmermon
70722,Jeffrey E Elbert
```

All terminal output is also saved to `solver_log.txt` — a plain text log of the full run, suitable for display in VBA or other integrations. The log path is overridable via `--log`.


> **Note:** If `schedule.csv` already exists it will be overwritten silently.

## Section Bucketing

Sections are split into three buckets after joining `sections.csv` and `time_blocks.csv` on CRN:

| Bucket | Criteria | Sent to OR | In Conflict Pairs |
|---|---|---|---|
| `regular` | Multi-day pattern (MWF, TuTh, MW, etc.) | ✓ | ✓ |
| `single_day` | Exactly one day (M, Tu, W, Th, F) | ✗ | ✗ |
| `internet` | No days/time value | ✓ | ✗ |

Internet section twin detection (same Sub + Num + Desc as an in-person section) is handled separately — see `KNOWN_ISSUES.md` KI-004.

## How to Invoke from Another Module

### Option A — one call, full pipeline

```python
import sys
sys.path.append("Solver")

from faculty_scheduling import main
main()
```

### Option B — step by step

```python
from ortools.sat.python import cp_model
from faculty_scheduling import load_all, build_csp, run_solver, validate, write_schedule_csv

# Step 1 — load CSVs
data = load_all(
    sections_path    = "sections.csv",
    time_blocks_path        = "time_blocks.csv",
    preferences_path = "preferences.csv",
    workload_path    = "workload.csv",   # optional
)

# Step 2 — build model
model = cp_model.CpModel()
built = build_csp(model, data)

# Step 3 — solve
assignment = run_solver(built)

# Step 4 — handle result
if assignment is None:
    print("No solution found.")
else:
    report = validate(assignment, data)
    report.print_report()
    write_schedule_csv(assignment, data, "schedule.csv")
```

The `assignment` dict maps CRN → faculty name:

```python
# {"76192": "Wendy Simmermon", "70722": "Alan Montgomery", ...}
for crn, faculty in assignment.items():
    print(f"CRN {crn} → {faculty}")
```

## Constraints

| ID | Type | Rule |
|---|---|---|
| C1 | Coverage | Every section in `sections_to_solve` assigned to exactly one faculty |
| C2 | Validity | All assigned faculty must exist in the faculty list |
| C3 | Workload | Each faculty's total units stay within their effective cap (soft) |
| C4 | Time Conflict | No faculty assigned to two time-overlapping regular sections |
| C5 | Exclusion | No `x`-marked faculty/section pair can be assigned |

**Objective:** Maximise the sum of `preference[section][faculty]` across all assignments, minus penalties for workload overload.

## Known Limitations

See `KNOWN_ISSUES.md` for the full list. Key items:

- **KI-001 — Dual credit sections** (e.g. CSC 461 / CSC 561 co-taught) are treated as independent sections and may cause false time conflict violations or split assignments.
- **KI-004 — Internet section twins** (online versions of in-person sections) are sent to OR without filtering. Twin detection based on Sub + Num + Desc is pending.
- **KI-003 — Room assignment** is not modelled. The `room` column is stored but not used in constraints.
- **Workload formula** — `current_workload` per section is currently a fixed value. Dynamic calculation based on enrollment, credits, and course level is planned.
