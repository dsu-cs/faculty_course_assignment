# Faculty Scheduling Solver

A constraint-based faculty scheduling tool built on [Google OR-Tools CP-SAT](https://developers.google.com/optimization/reference/python/sat/python/cp_model). Given faculty preferences, section time blocks, and workload limits, it finds the optimal faculty-to-section assignment that maximises preference scores while satisfying all constraints.

## Input CSV Formats

### `preferences.csv`
Rows are sections identified by CRN, columns are faculty names.

| section_id | Faculty A | Faculty B | Faculty C |
|---|---|---|---|
| 76192 | 3 | x | 1 |
| 70722 | 0 | 2 | x |

**Score values:**
- `0–3` — preference score (0 = neutral, 3 = strongly prefer)
- `x` — hard exclusion, this faculty **cannot** be assigned to this section regardless of score

The difference between `0` and `x` matters — a score of `0` means the faculty is indifferent but the solver can still assign them if needed. An `x` is an absolute ban the solver can never override.

---

### `time_blocks.csv`
One row per section. End time is derived automatically from the day pattern — do not include it.

| section_id | pattern | start_time | room |
|---|---|---|---|
| 76192 | TTh | 13:00 | _(leave blank)_ |
| 70722 | MWF | 14:00 | _(leave blank)_ |

**Supported day patterns and their fixed durations:**

| Pattern | Days | Duration |
|---|---|---|
| `MWF` | Mon / Wed / Fri | 50 min |
| `TTh` | Tue / Thu | 75 min |
| `MW` | Mon / Wed | 75 min |

The `room` column is reserved for future use — leave it blank for now.

---

### `workload.csv`
One row per faculty member.

| faculty | min_sections | max_sections |
|---|---|---|
| Wendy Simmermon | 1 | 3 |
| Jeffrey E Elbert | 1 | 2 |

## How to Run

### Full pipeline — uses default CSV filenames in the same folder
```bash
python faculty_scheduling.py
```

### Custom file paths
```bash
python faculty_scheduling.py \
  --preferences  data/preferences.csv \
  --time_blocks  data/time_blocks.csv \
  --workload     data/workload.csv \
  --output       output/schedule.csv
```

### Run the test suite — no CSV files needed
```bash
python faculty_scheduling.py --test
```

## Output

The solver writes `schedule.csv` — a 0/1 matrix where each row is a section
and each column is a faculty member. A `1` means that faculty is assigned to
that section.

```
section_id,Wendy Simmermon,Jeffrey E Elbert,...
76192,1,0,...
70722,0,1,...
```

It also prints a human-readable summary to the terminal:

```
══════════════════════════════════════════════════════════════
  SCHEDULE SUMMARY
══════════════════════════════════════════════════════════════
  Section        Assigned Faculty       Pref Score
  ────────────── ────────────────────   ──────────
  76192          Wendy Simmermon             +3
  70722          Alan Montgomery             +3
  ...
  ──────────────────────────────────────
  Total preference score..........      +52

  Faculty           Assigned  Min  Max  Status
  ────────────────────────────────────────────
  Wendy Simmermon          2    1    3  ✓ OK
  Jeffrey E Elbert         1    1    2  ✓ OK
```

> **Note:** If `schedule.csv` already exists it will be overwritten silently.

## How to Invoke from Another Module

### Option A — one call, full pipeline

```python
import sys
sys.path.append("Solver")

from faculty_scheduling import main
main()
```

### Option B — step by step, for integration into a larger app

```python
import sys
sys.path.append("Solver")

from ortools.sat.python import cp_model
from faculty_scheduling import load_all, build_csp, run_solver, validate, write_schedule_csv

# Step 1 — load CSVs into SchedulingData
data = load_all(
    preferences_path="Solver/preferences.csv",
    time_blocks_path="Solver/time_blocks.csv",
    workload_path="Solver/workload.csv",
)

# Step 2 — build the CSP model (not yet solved)
model = cp_model.CpModel()
built = build_csp(model, data)

# Step 3 — solve
assignment = run_solver(built)

# Step 4 — handle the result
if assignment is None:
    print("No solution found.")
else:
    report = validate(assignment, data)
    report.print_report()
    write_schedule_csv(assignment, data.sections, data.faculty, "Solver/schedule.csv")
```

The `assignment` dict can be used directly if you need the result in your own UI
rather than writing to CSV:

```python
# {"76192": "Wendy Simmermon", "70722": "Alan Montgomery", ...}
for crn, faculty in assignment.items():
    print(f"CRN {crn} → {faculty}")
```

## Constraints

| ID | Type | Rule |
|---|---|---|
| C1 | Coverage | Every section assigned to exactly one faculty |
| C2 | Validity | All assigned faculty must exist in the faculty list |
| C3 | Workload | Each faculty teaches between their `min` and `max` sections |
| C4 | Time Conflict | No faculty assigned to two time-overlapping sections |
| C5 | Exclusion | No `x`-marked faculty/section pair can be assigned |

**Objective:** Maximise the sum of `preference[section][faculty]` across all assignments.

Five tests run without any CSV files using hardcoded sample data:

| Test | What it checks |
|---|---|
| T1 | Conflict detection — day overlap and time interval logic |
| T2 | End time derivation — MWF / TTh / MW duration rules |
| T3 | Full pipeline — solver finds a valid, constraint-passing solution |
| T4 | Infeasibility detection — solver returns `None` when no solution exists |
| T5 | Exclusion constraint — `x`-marked assignments are never made |

```bash
python faculty_scheduling.py --test
```


## Known Limitations

See `KNOWN_ISSUES.md` for the full list. Key items:

- **Dual credit sections** (e.g. CSC 461 / CSC 561 co-taught) are treated as independent sections — will cause false time conflict violations or split assignments on the same faculty.
- **Online sections** (no days/times in the registrar data) are excluded from scheduling. They appear in `preferences.csv` but cannot be assigned.
- **Room assignment** is not modelled. The `room` column in `time_blocks.csv` is a placeholder for a future constraint.
